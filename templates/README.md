# <Project> README

This README is the routing map for the <Project> academic research folder.

## What Goes Where

| Lane | Purpose |
|---|---|
| `<Project>.md` | Obsidian landing page and high-level navigation |
| `RESEARCH.md` | Research question, contribution, claims, scope, study shape, target venue |
| `CURRENT_STATUS.md` | Current phase, priorities, blockers, recent progress, next actions |
| `literature/` | Paper notes, reading queue, related-work synthesis, citation gaps |
| `evidence/` | Source registry, data provenance, measurement definitions, data risks |
| `analysis/` | Methods, findings, interpretations, modeling decisions, methodology/process audits |
| `verification/` (optional) | Data verification reports, reproducibility checks, hand-calculation logs, cross-file consistency audits |
| `writing/` | Outline, draft status, figures, tables, submission/revision notes |
| `meetings/` | Advisor/collaborator meeting notes, feedback, action items |
| `planning/` | Concrete work plans and research decisions |
| `history/` | Concise completed-work logs |
| `archive/` | Superseded notes and retired drafts |

## Optional Lanes

| Lane | Purpose |
|---|---|
| `verification/` | Data verification reports, reproducibility checks, hand-calculation logs, cross-file consistency audits |
| `submissions/` | Submission packages, cover letters, reviewer responses |
| `admin/` | Funding, travel, compliance, deadlines |
| `ethics/` | IRB, consent, privacy, human-subjects documentation |
| `collaboration/` | Shared notes with co-authors |

## Update Rules

- Read this README before routing project information.
- Update current-state notes before history.
- Keep history concise; move detailed reports to `analysis/` or `verification/`.
- Update folder indexes whenever notes are added, moved, archived, or deleted.
- Do not store raw sensitive data in this PM folder unless this README explicitly permits it.
